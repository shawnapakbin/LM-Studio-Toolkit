# BlenderBridge

AI-driven 3D content creation and Blender automation through the LLM Toolkit's MCP infrastructure. BlenderBridge provides orchestration tools that compose multiple Blender MCP operations into streamlined workflows.

## Prerequisites

Complete these steps in order before using BlenderBridge:

1. **Install Blender 5.1+**
   Download and install Blender 5.1 or later from [blender.org](https://www.blender.org/download/).

2. **Install the Blender MCP add-on**
   Download the latest release from [projects.blender.org/lab/blender_mcp](https://projects.blender.org/lab/blender_mcp) releases page. Install via drag-and-drop of the `.zip` file onto the Blender window (or use Edit → Preferences → Add-ons → Install from Disk).

3. **Open Blender with a new/blank project**
   Launch Blender and start with a new, blank project (File → New → General). Existing scenes may produce unpredictable results with LLM-generated operations.

4. **Confirm add-on is listening on localhost:9876**
   Open the 3D Viewport N-panel (press `N`), navigate to the **MCP** tab, and verify the status shows "Listening on 127.0.0.1:9876".

5. **Install the MCP server binary**
   Install the `blender-mcp` binary via the `.mcpb` bundle from the [official releases](https://projects.blender.org/lab/blender_mcp/releases) or build from source. Ensure the binary is available on your system PATH.

## Security Warning

> **⚠️ The MCP server executes LLM-generated code in Blender without guards.** Use a system without access to sensitive information. The Blender MCP server runs arbitrary Python code generated by the LLM directly inside your Blender instance. Do not use this on a machine containing sensitive data, credentials, or production assets.

## Usage

### Verify Setup

Run the setup verification script to confirm everything is connected:

```bash
npm run setup:blender
```

This checks the TCP connection to the Blender add-on and outputs a configuration summary on success.

### Available Tools

| Tool | Description |
|------|-------------|
| `blender_health_check` | Verifies Blender add-on connectivity and MCP server process |
| `blender_create_object` | Creates a named object with geometry type and transforms |
| `blender_scene_summary` | Returns scene hierarchy, active object, and render settings |
| `blender_render_preview` | Renders a 480×270 PNG preview thumbnail |
| `blender_export_to_viewer` | Exports the active object as OBJ to the system temp directory |

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `BLENDER_MCP_HOST` | `127.0.0.1` | Blender add-on TCP host |
| `BLENDER_MCP_PORT` | `9876` | Blender add-on TCP port |
| `BLENDER_MCP_COMMAND` | `blender-mcp` | MCP server binary name/path |
| `BLENDER_MCP_ARGS` | _(empty)_ | Additional arguments for the MCP server binary |

## Troubleshooting

### Add-on not enabled

**Symptom:** Health check returns `BLENDER_ADDON_UNREACHABLE` or the N-panel MCP tab is missing.

**Resolution:**
1. Open Blender and go to Edit → Preferences → Add-ons.
2. Search for "MCP" in the add-on list.
3. Enable the Blender MCP add-on by checking its checkbox.
4. Open the 3D Viewport N-panel (press `N`) and confirm the MCP tab shows "Listening on 127.0.0.1:9876".

### Blender not open

**Symptom:** Health check returns `BLENDER_ADDON_UNREACHABLE` with "Connection refused".

**Resolution:**
1. Launch Blender 5.1 or later.
2. Open a new/blank project (File → New → General).
3. Verify the MCP add-on is enabled and listening (see "Add-on not enabled" above).
4. Re-run the health check.

### Port conflict on 9876

**Symptom:** Add-on shows an error about the port being in use, or health check connects but gets unexpected responses.

**Resolution:**
1. Identify what process is using port 9876:
   - Windows: `netstat -ano | findstr :9876`
   - macOS/Linux: `lsof -i :9876`
2. Stop the conflicting process, or configure a different port:
   - Set the port in the Blender MCP add-on preferences.
   - Set the `BLENDER_MCP_PORT` environment variable to match.

### Firewall blocking localhost

**Symptom:** Health check times out after 5 seconds with `BLENDER_ADDON_UNREACHABLE`, even though Blender is open and the add-on shows "Listening".

**Resolution:**
1. Check that your firewall allows localhost (127.0.0.1) TCP connections on port 9876.
2. On Windows, add an inbound rule in Windows Firewall for port 9876.
3. On macOS, check System Settings → Network → Firewall and allow connections for Blender.
4. Temporarily disable the firewall to confirm it's the cause, then add the appropriate rule.
